Актуальность темы

Современная система образования переживает период цифровой трансформации. Развитие технологий искусственного интеллекта, облачных вычислений и мобильных приложений расширяет возможности организации учебного процесса. По данным отчёта UNESCO «AI and Education: Guidance for Policy-makers» (2021), внедрение AI-технологий в образование помогает адаптировать учебный процесс под каждого ученика и повышать академические результаты.

Отдельного внимания заслуживает разработка образовательных инструментов для школьников младших и средних классов. Эта возрастная группа активно осваивает цифровые технологии, но нуждается в адаптированном и безопасном образовательном пространстве. По данным ежегодного отчёта «Лаборатории Касперского» (2024), дети проводят в интернете в среднем 4–6 часов ежедневно, поэтому создание для них полезного и безопасного цифрового контента — задача первостепенной важности.

Мессенджер Telegram насчитывает более 900 миллионов активных пользователей и давно стал привычной средой общения подростков. Telegram Bot API и технология Mini App позволяют создавать полноценные интерактивные приложения внутри мессенджера, и пользователю не приходится устанавливать отдельное приложение. Это делает Telegram удобной платформой для размещения образовательных сервисов.

Однако существующие образовательные решения на базе AI часто ориентированы на взрослую аудиторию, не учитывают возрастные особенности восприятия информации и не дают необходимого уровня модерации контента для детей. Большинство таких решений ограничивается текстовым чатом и не предлагает мультимодальных возможностей: распознавание голоса, анализ изображений, визуализации, образовательные игры.

Разработка образовательной платформы, объединяющей AI, мультимодальный ввод и геймификацию в безопасной среде для школьников, остаётся нерешённой задачей с высоким прикладным значением.

Проблема проекта

На сегодняшний день отсутствует доступная, безопасная и комплексная образовательная платформа для школьников, которая объединяла бы искусственный интеллект с интерактивными методами обучения.

«Сущее» (то, что есть):
* Существующие AI-ассистенты (ChatGPT, Gemini) не адаптированы для детской аудитории и не имеют встроенной модерации контента.
* Школьные образовательные платформы не предлагают мультимодальных возможностей: голосовые вопросы, проверка домашних заданий по фото, генерация визуализаций.
* Отсутствует интеграция AI-обучения с элементами геймификации (игры, достижения, прогресс), которые способны значительно повысить мотивацию к обучению.
* Не существует образовательной платформы, доступной непосредственно в Telegram — привычной среде общения для подростков.

«Должное» (то, что необходимо):
* Необходимо разработать платформу с AI-чатом, адаптирующим ответы по возрасту и классу ученика.
* Требуется реализовать мультимодальный ввод: текст, голос, фото.
* Важно обеспечить многоуровневую модерацию контента для гарантии безопасности детей.
* Необходимо внедрить элементы геймификации для повышения вовлечённости в учебный процесс.

Объект и предмет исследования

Объект исследования — процесс цифровизации школьного образования с применением технологий искусственного интеллекта.

Предмет исследования — методы и средства разработки образовательной платформы на базе Telegram Bot API с использованием больших языковых моделей, семантического поиска и геймификации для повышения эффективности обучения школьников 1–9 классов.

Цель проекта

Цель проекта — разработать и развернуть в продуктивной среде полнофункциональную образовательную платформу PandaPal в формате Telegram-бота и веб-приложения, обеспечивающую школьникам 1–9 классов доступ к AI-помощнику с мультимодальными возможностями, образовательными играми, системой геймификации и многоуровневой модерацией контента.

Для достижения поставленной цели необходимо решить следующие задачи:

4. Разработка клиентской части:
   * Создать веб-приложение (Mini App) на React 19 с TypeScript 5 и Tailwind CSS 3.4.

Питомец мотивирует ребёнка возвращаться к обучению, формируя эмоциональную привязанность и привычку ежедневных занятий.

ГЛАВА 1. ТЕОРЕТИЧЕСКИЕ ОСНОВЫ РАЗРАБОТКИ ОБРАЗОВАТЕЛЬНЫХ ПЛАТФОРМ С ИСКУССТВЕННЫМ ИНТЕЛЛЕКТОМ


1.1. Искусственный интеллект в современном образовании

Искусственный интеллект (ИИ) — направление компьютерных наук, занимающееся созданием систем, решающих задачи, которые ранее требовали участия человека: понимание текста, распознавание образов, принятие решений. С появлением больших языковых моделей (Large Language Models, LLM) ИИ перестал быть инструментом узких специалистов и вошёл в повседневную практику.

Применение ИИ в образовании охватывает несколько ключевых направлений:

1. Персонализированное обучение. AI-системы анализируют уровень знаний учащегося, темп усвоения материала и предпочитаемый стиль подачи информации, формируя на этой основе индивидуальные образовательные траектории. В мета-анализе Zawacki-Richter et al. (2019), охватившем 146 работ, зафиксировано повышение академической успеваемости на 20–30 % при использовании персонализированных AI-инструментов.

2. Автоматизированная оценка и обратная связь. AI позволяет автоматизировать проверку заданий, предоставляя учащимся мгновенную обратную связь. Это особенно важно для предметов с однозначными ответами (математика, тесты), но современные LLM способны оценивать и творческие работы.

3. Интеллектуальные помощники и чат-боты. Образовательные чат-боты выступают в роли виртуальных репетиторов, доступных круглосуточно. Они могут отвечать на вопросы, объяснять темы, генерировать задачи и направлять учащегося в процессе изучения нового материала.

4. Мультимодальное взаимодействие. Современные AI-системы поддерживают не только текстовый ввод, но и голосовые запросы, анализ изображений (например, фото рукописного задания), генерацию визуализаций (графиков, диаграмм, схем), что обеспечивает более естественное и эффективное взаимодействие.

Среди конкретных технологий, применяемых в образовательном ИИ, выделяются большие языковые модели (GPT, YandexGPT, GigaChat), системы распознавания речи (Speech-to-Text), оптического распознавания текста (OCR), генерации изображений и векторных эмбеддингов для семантического поиска.

Для PandaPal выбрана модель YandexGPT — российская LLM, разработанная компанией Яндекс. Она поддерживает русский язык на высоком уровне, имеет возможность настройки параметров генерации (temperature, max_tokens) и предоставляется через облачный API Yandex Cloud, что подходит для русскоязычной образовательной платформы.


1.2. Чат-боты как инструмент обучения: обзор существующих решений

Чат-бот — программа, использующая обработку естественного языка (Natural Language Processing, NLP) для ведения диалога с пользователем в текстовом или голосовом формате. В сфере образования чат-боты играют роль виртуальных репетиторов: отвечают на вопросы, разъясняют материал и помогают выстроить траекторию обучения.

Различают два основных типа образовательных чат-ботов:

1. Чат-боты на основе правил (rule-based). Работают по заранее заданным сценариям и шаблонам. Способны обрабатывать ограниченный набор запросов. Примеры: боты-навигаторы по расписанию, боты с тестами.

2. Чат-боты на основе AI (AI-powered). Используют большие языковые модели для генерации ответов. Способны вести свободный диалог, понимать контекст, адаптировать стиль общения. Примеры: ChatGPT, Claude, Gemini, YandexGPT.

Обзор существующих образовательных платформ с AI:

* ChatGPT (OpenAI) — мощная языковая модель, широко используемая для образовательных целей. Недостатки: не адаптирована для детей, отсутствие встроенной модерации для детского контента, нет интеграции с Telegram, ориентация на англоязычную аудиторию.

* Duolingo — платформа для изучения языков с элементами геймификации. Достоинства: эффективная геймификация, персонализация. Недостатки: ограничена одним предметом (иностранные языки), не использует чат-формат.

* Учи.ру — российская образовательная платформа для школьников. Достоинства: адаптация под российскую программу. Недостатки: ограниченные AI-возможности, отсутствие мультимодального ввода.

* Яндекс Учебник — образовательные материалы от Яндекса. Достоинства: качественный контент, интеграция с российской программой. Недостатки: не является чат-ботом, нет голосового ввода и анализа фото.

Платформа PandaPal объединяет преимущества перечисленных решений: AI-чат с адаптацией по возрасту (как ChatGPT, но безопаснее), геймификация (как Duolingo), ориентация на российскую программу (как Учи.ру) и интеграция с экосистемой Yandex Cloud. При этом PandaPal размещается непосредственно в Telegram, что упрощает доступ для целевой аудитории.

Согласно обзору Hwang & Chang (2021), чат-боты в образовании повышают мотивацию учащихся и снижают тревожность при обращении за помощью, одновременно обеспечивая круглосуточную доступность ресурса. При этом авторы подчёркивают вспомогательную роль ИИ: AI должен поддерживать обучение, а не подменять его.


1.3. Технология Retrieval-Augmented Generation (RAG) и её применение в образовании

Retrieval-Augmented Generation (RAG) — архитектурный паттерн, при котором LLM дополняется результатами поиска по внешней базе знаний. RAG-система сначала находит релевантные документы, а затем передаёт их модели как контекст для генерации ответа, благодаря чему модель опирается не только на обучающие данные, но и на актуальные источники.

Архитектура RAG включает три основных компонента:

1. Хранилище знаний. Коллекция документов, статей, учебных материалов, индексированных для эффективного поиска. В PandaPal используется PostgreSQL с расширением pgvector, позволяющим хранить и искать векторные представления текстов.

2. Модель эмбеддингов. Нейросетевая модель, преобразующая текстовые фрагменты в числовые векторы (эмбеддинги), которые отражают семантическое значение текста. В PandaPal используется Yandex Embeddings API.

3. Генеративная модель. Большая языковая модель, получающая найденные документы как контекст и генерирующая ответ на основе этого контекста. В PandaPal используется YandexGPT Pro.

Процесс работы RAG в PandaPal:
1. Пользователь задаёт вопрос (например, «Что такое фотосинтез?»).
2. Вопрос преобразуется в вектор с помощью Yandex Embeddings API.
3. Выполняется поиск ближайших по смыслу документов в таблице knowledge_embeddings (PostgreSQL + pgvector) с использованием косинусного расстояния.
4. Найденные фрагменты передаются вместе с вопросом в YandexGPT Pro.
5. Модель генерирует ответ, опираясь на достоверные данные из базы знаний.

Преимущества RAG в образовательном контексте:
* Снижение галлюцинаций. Модель опирается на проверенные источники, а не генерирует факты произвольно. В образовательном контексте это особенно существенно.
* Актуальность информации. Базу знаний можно обновлять без переобучения модели.
* Прозрачность. Ответ может содержать ссылки на конкретные источники из базы знаний.
* Контролируемость. Администратор решает, какие материалы попадают в базу.

В PandaPal RAG-пайплайн дополнен несколькими оптимизациями:
* Расширение запросов (Query Expansion) — автоматическое обогащение пользовательского запроса синонимами и связанными терминами для повышения полноты поиска.
* Семантический кэш — кэширование результатов поиска для семантически похожих запросов, что снижает нагрузку на API эмбеддингов.
* Компрессия контекста — сжатие найденных документов для оптимального использования контекстного окна LLM.
* Ранжирование результатов (Reranker) — повторное ранжирование найденных документов по релевантности перед передачей в модель.


1.4. Геймификация учебного процесса: теория и практика

Геймификация (gamification) — это применение элементов игрового дизайна в неигровых контекстах с целью повышения вовлечённости, мотивации и активности пользователей. В образовании геймификация предполагает внедрение таких элементов, как баллы (XP), уровни, достижения (бейджи), рейтинги и соревнования в учебный процесс.

Теоретические основы геймификации опираются на ряд психологических теорий:

1. Теория самодетерминации (Self-Determination Theory, Deci & Ryan). Выделяет три базовые потребности человека: автономия, компетентность и связанность. Геймификация удовлетворяет все три: пользователь сам выбирает задачи (автономия), получает обратную связь о своём прогрессе (компетентность), может видеть рейтинги других (связанность).

2. Теория потока (Flow Theory, Csikszentmihalyi). Описывает состояние максимальной вовлечённости, возникающее при балансе между сложностью задачи и навыками пользователя. Геймификация способствует достижению потокового состояния через адаптивную сложность.

3. Теория подкрепления (Reinforcement Theory, Skinner). Положительное подкрепление (награды, баллы, достижения) закрепляет желаемое поведение — в данном случае активное обучение.

В мета-анализе Sailer & Homner (2020) показано, что геймификация увеличивает вовлечённость учащихся в среднем на 25 %. В отечественном исследовании Кречетовой и др. (2023) средний балл учащихся вырос с 65,4 до 78,9 после внедрения геймифицированных элементов, а 92 % педагогов отметили повышение мотивации.

В PandaPal система геймификации реализована через несколько механизмов:

* Очки опыта (XP) — начисляются за отправку сообщений, решение задач, участие в играх.
* Достижения — система из более чем 15 бейджей, разблокируемых при достижении определённых показателей (первое сообщение, 100 сообщений, неделя подряд, вопросы по пяти и более предметам).
* Образовательные игры PandaPalGo — (5 игр: тамагочи «Моя панда», крестики-нолики, шашки, 2048, эрудит), развивающие логику, стратегическое мышление и словарный запас.
* «Моя панда» — виртуальный питомец (тамагочи), который меняет настроение и активность в зависимости от того, насколько регулярно ученик пользуется платформой. Этот механизм создаёт эмоциональную привязанность и мотивирует к ежедневному обучению.
* Прогресс и статистика — пользователь видит свой прогресс по предметам, количество решённых задач, дни активности.


1.5. Безопасность детей в цифровой среде: модерация и фильтрация контента

Информационная безопасность детей в интернете — вопрос, требующий системного подхода. В Российской Федерации данная сфера регулируется Федеральным законом № 436-ФЗ «О защите детей от информации, причиняющей вред их здоровью и развитию», который устанавливает требования к контентной фильтрации в образовательных учреждениях и на платформах, ориентированных на несовершеннолетних пользователей.

При разработке образовательной платформы с AI для детей особое внимание необходимо уделить следующим аспектам безопасности:

1. Фильтрация входящего контента. Пользовательские запросы (текст, голос, изображения) должны проверяться на наличие недопустимого содержания до обработки AI-моделью. Тем самым предотвращаются попытки получить небезопасный контент через манипуляцию промптами (prompt injection).

2. Модерация выходного контента. Ответы AI-модели также требуют проверки, поскольку языковые модели могут генерировать неуместную или некорректную информацию. Многоуровневая фильтрация на выходе добавляет дополнительный рубеж защиты.

3. Адаптация контента по возрасту. Ответы должны учитывать возраст и класс ученика, используя соответствующую лексику, сложность объяснений и примеры.

В PandaPal модерация контента реализована как многоуровневая система:

* Уровень 1: Фильтрация запрещённых паттернов (forbidden patterns). Проверка входящих сообщений на наличие нецензурной лексики, запрещённых тем и опасного контента с использованием регулярных выражений и словарей.

* Уровень 2: Расширенная модерация (advanced moderation). Анализ контекста сообщения для выявления скрытых попыток обхода фильтров, использование эвристик и лингвистического анализа.

* Уровень 3: AI-модерация. Использование языковой модели для семантического анализа сообщений, которые прошли первые два уровня, но могут содержать неуместный контент в неявной форме.

* Уровень 4: Сервис взрослых тем (adult topics service). Специализированный модуль для корректной обработки вопросов на чувствительные темы (здоровье, семья), которые не являются запрещёнными, но требуют деликатного подхода с учётом возраста.

* Адаптивный промпт. Системный промпт для YandexGPT содержит инструкции по адаптации ответов в зависимости от возраста и класса ученика, запрет на определённые темы и указание стилистических характеристик ответов.


1.6. Обзор используемых технологий и обоснование выбора стека

При выборе стека PandaPal учитывались конкретные требования: качественная поддержка русского языка в AI-моделях, совместимость с Telegram Bot API, производительность и простота деплоя.

Серверная часть (Backend):

* Python 3.13 — основной язык разработки. Выбран за широкую экосистему библиотек для AI и веб-разработки, простоту синтаксиса и активное сообщество. Версия 3.13 содержит улучшения производительности.

* aiogram 3.24 — асинхронный фреймворк для создания Telegram-ботов. Выбран как наиболее производительный и функциональный из доступных библиотек для Python, с полной поддержкой асинхронного программирования.

ГЛАВА 2. ПРАКТИЧЕСКАЯ ЧАСТЬ: АРХИТЕКТУРА И РЕАЛИЗАЦИЯ ПЛАТФОРМЫ PANDAPAL


2.1. Общая архитектура системы

Платформа PandaPal построена по клиент-серверной архитектуре с множеством внешних интеграций. Общий поток данных можно описать следующей схемой:

```
Пользователь → Telegram / Браузер
                     ↓
            Telegram Bot API / HTTPS
                     ↓
        ┌────────────────────────────┐
        │    web_server.py           │
        │    (точка входа)           │
        │                            │
        │  ┌──────────┐ ┌─────────┐ │
        │  │ aiogram   │ │ aiohttp │ │
        │  │ (Bot API) │ │ (REST)  │ │
        │  └──────────┘ └─────────┘ │
        └────────────┬───────────────┘
                     ↓
        ┌────────────────────────────┐
        │       bot/services/        │
        │    (51 модуль)             │
        │                            │
        │  AI · RAG · Модерация      │
        │  Игры · Платежи · Кэш     │
        │  Визуализация · Аналитика  │
        └────────────┬───────────────┘
                     ↓
    ┌────────┬───────┴───────┬──────────┐
    │        │               │          │
PostgreSQL  Redis      Yandex Cloud  YooKassa
+ pgvector  (Upstash)  (GPT, STT,   (платежи)
                        OCR, ART)
```

Серверная часть реализована как единое Python-приложение (`web_server.py`), которое одновременно обрабатывает webhook-запросы от Telegram Bot API и REST-запросы от веб-приложения через aiohttp. Благодаря этому оба канала — Telegram и браузер — обслуживаются одним бэкендом.

Ключевые архитектурные решения:

1. Модульная структура сервисов. Каталог `bot/services/` содержит 51 модуль бизнес-логики, каждый из которых отвечает за конкретную предметную область: `yandex_cloud_service.py` — взаимодействие с Yandex Cloud API; `knowledge_service.py` — управление базой знаний; `moderation_service.py` — модерация контента; `games_service/` — логика образовательных игр; `payment_service.py` — интеграция с YooKassa; `visualization_service.py` — генерация визуализаций.

2. Слой ORM-моделей. Каталог `bot/models/` содержит 10 моделей SQLAlchemy, описывающих сущности предметной области: User (пользователь), Chat (сообщения чата), Learning (учебные сессии), Games (игровые сессии), Payments (платёжные транзакции), PandaPet (виртуальный питомец), Referral (реферальная программа), Analytics (аналитические данные).

3. REST API. Каталог `bot/api/` содержит эндпоинты для Mini App и веб-приложения, следуя RESTful-принципам. Все защищённые эндпоинты валидируют владельца ресурса через заголовок `X-Telegram-Init-Data`.

4. Middleware и безопасность. Каталоги `bot/middleware/` и `bot/security/` реализуют сквозные механизмы: rate limiting, аудит запросов, криптографическая проверка Telegram-данных, валидация входных данных через Pydantic v2.


2.2. Серверная часть: Python, aiogram, SQLAlchemy

Серверная часть PandaPal написана на Python 3.13 с применением async/await, что позволяет обрабатывать множество запросов параллельно без блокировки потока.

Файл `web_server.py` (точка входа, ~25 КБ) создаёт и настраивает два основных компонента:

1. aiogram Bot и Dispatcher — для обработки Telegram-сообщений через webhook. Бот принимает текстовые, голосовые, фото- и документальные сообщения, обрабатываемые соответствующими handlers в `bot/handlers/`.

2. aiohttp Application — для обслуживания REST API запросов от Mini App и веб-приложения. Маршруты определены в `server_routes/` и `bot/api/`.

Пример обработки текстового сообщения:
```
Пользователь отправляет сообщение
         ↓
bot/handlers/ai_chat/text.py → получение текстового сообщения
         ↓
bot/services/moderation_service.py → проверка контента
         ↓
bot/services/prompt_builder.py → формирование промпта
         ↓
bot/services/knowledge_service.py → RAG-поиск (если нужен)
         ↓
bot/services/yandex_cloud_service.py → запрос к YandexGPT
         ↓
bot/services/yandex_ai_response_generator.py → обработка ответа
         ↓
Отправка ответа пользователю
```

Слой работы с базой данных построен на SQLAlchemy 2.0 в асинхронном режиме (драйвер psycopg 3). Использование ORM с параметризованными запросами исключает SQL-инъекции. Миграции схемы ведутся через Alembic.

Пример ORM-модели пользователя (упрощённо):
```python
class User(Base):
    __tablename__ = "users"

    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    telegram_id: Mapped[int] = mapped_column(
        BigInteger, unique=True, nullable=False, index=True
    )
    username: Mapped[str | None] = mapped_column(String(255), nullable=True)
    first_name: Mapped[str | None] = mapped_column(String(255), nullable=True)
    age: Mapped[int | None] = mapped_column(Integer, nullable=True)
    grade: Mapped[int | None] = mapped_column(Integer, nullable=True)
    user_type: Mapped[str] = mapped_column(
        String(20), nullable=False, default="child"
    )
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), server_default=func.now()
    )
    premium_until: Mapped[datetime | None] = mapped_column(
        DateTime(timezone=True), nullable=True
    )

    @property
    def is_premium(self) -> bool:
        """Активна ли Premium подписка."""
        ...
```

В проекте используется современный стиль SQLAlchemy 2.0 с аннотациями типов (`Mapped[]` и `mapped_column()`), что обеспечивает строгую типизацию на уровне Python. Поле `is_premium` реализовано как вычисляемое свойство (`@property`), определяющее статус подписки по дате `premium_until` и наличию активных подписок.


2.3. Интеграция с YandexGPT и формирование AI-ответов

Ключевая интеграция платформы — взаимодействие с YandexGPT, LLM от Яндекса. Для взаимодействия с моделью реализован сервис `yandex_cloud_service.py` (~43 КБ), который инкапсулирует всю логику работы с Yandex Cloud API.

Формирование AI-ответа включает несколько этапов:

1. Построение промпта (`prompt_builder.py`, ~16 КБ). Системный промпт формируется динамически в зависимости от:
   * Возраста и класса пользователя — определяет сложность языка и объём объяснений.
   * Выбранного предмета — добавляет предметно-специфические инструкции.
   * Контекста предыдущих сообщений — обеспечивает связность диалога.
   * Результатов RAG-поиска — включает релевантные фрагменты из базы знаний.

2. Формирование контекста (`ai_context_builder.py`, ~12 КБ). Модуль управляет историей диалога, ограничивая её 200 сообщениями. При достижении 400 сообщений автоматически удаляются старейшие записи. История помогает модели поддерживать связность ответов.

3. Генерация ответа. Запрос отправляется к YandexGPT Pro через REST API Yandex Cloud с параметрами: `temperature=0.35` (низкая «температура» для более точных ответов), `max_tokens=8192`.

4. Обработка ответа (`yandex_ai_response_generator.py`, ~22 КБ). Модуль поддерживает два режима генерации:
   * Обычный (non-streaming) — ответ получается целиком после завершения генерации.
   * Потоковый (streaming, SSE) — ответ передаётся по частям по мере генерации, обеспечивая мгновенную обратную связь пользователю.

5. Постобработка ответа. Включает очистку от нежелательных артефактов (`response_cleaner/`), проверку формул (`formula_explainer.py`), генерацию визуализаций при необходимости (`visualization_service.py`).

Помимо YandexGPT, платформа интегрирована с другими сервисами Yandex Cloud:
* SpeechKit STT (`speech_service.py`) — распознаёт голосовые сообщения пользователей, преобразуя их в текст для последующей обработки AI.
* Vision OCR (`vision_service.py`) — распознаёт текст на фотографиях, что позволяет проверять домашние задания по фото.
* YandexART (`yandex_art_service.py`) — генерирует изображения по текстовому описанию для визуализации учебных тем.


2.4. RAG-пайплайн: семантический поиск по базе знаний

RAG-пайплайн PandaPal реализован в каталоге `bot/services/rag/` и состоит из пяти основных компонентов:

1. Векторный поиск (`vector_search.py`, ~8.5 КБ). Выполняет поиск ближайших векторов в таблице `knowledge_embeddings` с использованием pgvector. Применяет косинусное расстояние для определения семантической близости. Возвращает top-k наиболее релевантных документов.

2. Расширение запросов (`query_expander.py`, ~26 КБ). Автоматически обогащает пользовательский запрос синонимами, связанными терминами и альтернативными формулировками. Приём повышает полноту поиска — в первую очередь для коротких и неточных запросов, типичных для детской аудитории.

3. Ранжирование (`reranker.py`, ~7.7 КБ). После первичного векторного поиска выполняет повторное ранжирование результатов по релевантности, учитывая не только семантическую близость, но и контекстуальные факторы.

4. Компрессия контекста (`compressor.py`, ~4.2 КБ). Сжимает найденные документы до оптимального размера для контекстного окна LLM, удаляя избыточную информацию и сохраняя ключевые факты.

5. Семантический кэш (`semantic_cache.py`, ~7.7 КБ). Кэширует результаты поиска для семантически похожих запросов. Если новый запрос похож на предыдущий (по косинусному расстоянию эмбеддингов), результат возвращается из кэша без обращения к базе данных, что снижает нагрузку и ускоряет ответ.

Наполнение базы знаний осуществляется через скрипт `scripts/update_knowledge_base.py`, который индексирует учебные материалы и энциклопедические источники, преобразуя их в эмбеддинги через Yandex Embeddings API и сохраняя в таблицу `knowledge_embeddings`.


2.5. Клиентская часть: React, TypeScript, Telegram Mini App

Клиентская часть PandaPal — одностраничное приложение (SPA) на React 19 с TypeScript 5, собираемое Vite 7. Приложение функционирует в двух режимах:

1. Telegram Mini App — запускается внутри мессенджера Telegram через кнопку бота. Использует Telegram Mini App SDK 8.0 для интеграции с интерфейсом мессенджера (цветовая тема, haptic feedback, кнопка «Назад»).

2. Веб-приложение — доступно по адресу https://pandapal.ru через любой браузер. Обеспечивает полный функционал платформы без необходимости использования Telegram.

Структура фронтенда организована по функциональному принципу (feature-based architecture):

```
frontend/src/
├── features/           # Функциональные модули

│   │   ├── Game2048.tsx
```

Серверное состояние управляется через TanStack Query 5 (кэширование запросов, фоновая синхронизация, оптимистичные обновления). Локальное состояние управляется через Zustand 5 — легковесную библиотеку с минимальным boilerplate.

AI-чат поддерживает streaming через Server-Sent Events (SSE): ответ появляется на экране по мере генерации, аналогично интерфейсу ChatGPT. Ученик видит текст сразу, а не ждёт несколько секунд, пока модель закончит генерацию целиком.


2.6. Система модерации контента

Модерация в PandaPal устроена как многоуровневый конвейер: каждое сообщение проходит несколько ступеней проверки. Основные модули:

* `moderation_service.py` (~20 КБ) — основной сервис базовой модерации. Содержит словари запрещённых паттернов, регулярные выражения для обнаружения нецензурной лексики и опасных тем.

* `advanced_moderation.py` (~21 КБ) — расширенная модерация. Анализирует контекст сообщения, выявляет попытки обхода фильтров (замена букв символами, транслитерация, использование эвфемизмов).

* `ai_moderator.py` (~2.4 КБ) — AI-модерация. Использует YandexGPT для семантического анализа сообщений, которые прошли предыдущие уровни, но могут содержать неуместный контент.

* `adult_topics_service.py` (~7.4 КБ) — обработка чувствительных тем. Определяет вопросы на деликатные темы (здоровье, семейные отношения) и формирует соответствующие инструкции для AI, обеспечивая корректный и деликатный ответ с учётом возраста.

Результат модерации определяет дальнейшую обработку сообщения: допуск к AI-обработке, блокировка с уведомлением пользователя или передача администратору для ручной проверки.


2.7. Образовательные игры и геймификация

В PandaPal реализовано 5 образовательных игр, объединённых под брендом PandaPalGo:

1. «Моя панда» (тамагочи) — виртуальный питомец-панда, которого нужно кормить, укладывать спать, играть с ним. Состояние панды зависит от регулярности использования платформы. Реализация: фронтенд `MyPanda/`, бэкенд `panda_pet_service.py` (~14 КБ), эндпоинты `panda_pet_endpoints.py` (~9 КБ).

2. Крестики-нолики (TicTacToe) — классическая игра с AI-противником. Реализация: `TicTacToe.tsx` (~10 КБ), серверная логика в `game_engines/`.

3. Шашки (Checkers) — стратегическая настольная игра с AI-противником, несколько уровней сложности. Реализация: `Checkers.tsx` (~16 КБ).

4. 2048 — числовая головоломка, развивающая логическое мышление. Реализация: `Game2048.tsx` (~16 КБ).

5. Эрудит (Erudite) — словесная игра, расширяющая словарный запас. Использует словарь из `erudite_dictionary.py` (~14 КБ). Реализация: `Erudite.tsx` (~13 КБ).

Серверная логика игр реализована в `bot/services/games_service/` и `bot/services/game_engines/`, а API-эндпоинты — в `bot/api/games_endpoints.py` (~21 КБ). Все игровые сессии сохраняются в базе данных с привязкой к пользователю, что позволяет отслеживать прогресс и начислять XP.

Система геймификации (`gamification_service.py`, ~31 КБ) управляет:
* Начислением XP за различные действия (сообщения, решённые задачи, победы в играх).
* Разблокировкой достижений при достижении определённых условий.
* Расчётом уровня пользователя на основе накопленного XP.
* Формированием статистики прогресса для отображения на дашборде.


2.8. Premium-подписка и платёжная система

PandaPal предлагает Premium-подписку стоимостью 299 ₽/месяц, расширяющую возможности платформы: неограниченное количество AI-запросов, приоритетная обработка, расширенные визуализации, дополнительный контент по предметам.

Интеграция с платёжной системой YooKassa реализована через несколько модулей:

* `payment_service.py` (~19 КБ) — основной сервис обработки платежей. Создание платёжных ссылок, обработка webhook-уведомлений от YooKassa, обновление статуса подписки.

* `subscription_service.py` (~11 КБ) — управление подписками. Активация, продление, отмена подписки, проверка статуса.

* `recurring_payment_service.py` (~7 КБ) — автоматическое продление подписки через сохранённые платёжные методы.

* `premium_features_service.py` (~20 КБ) — логика Premium-функций. Определение доступного функционала в зависимости от статуса подписки.

* `premium_endpoints.py` (~32 КБ) — REST API для работы с Premium из Mini App.

Дополнительно реализована реферальная программа (`referral_service.py`): преподаватели и партнёры получают персональную ссылку, и оплаты, совершённые по этой ссылке, учитываются в системе выплат.


2.9. Развёртывание и обеспечение непрерывной доставки

Платформа PandaPal развёрнута в продуктивной среде на платформе Railway — PaaS-решении, поддерживающем контейнеризацию через Docker, управляемые базы данных и автоматическое масштабирование. Выбор Railway обусловлен простотой деплоя, встроенной поддержкой переменных окружения и отсутствием необходимости в ручном администрировании серверов.

Контейнеризация и Dockerfile. Серверная часть упакована в Docker-контейнер на основе образа `python:3.13-slim`. Dockerfile включает установку системных зависимостей, копирование исходного кода, установку Python-зависимостей через `pip` и определение точки входа (`web_server.py`). Многоступенчатая сборка минимизирует размер финального образа.

Конвейер CI/CD реализован через GitHub Actions и включает следующие этапы:

1. Линтинг и статический анализ. При каждом push и pull request автоматически запускаются проверки: ESLint и TypeScript strict mode для фронтенда, Bandit для анализа безопасности Python-кода.

2. Автоматизированное тестирование. Запускаются unit-тесты и интеграционные тесты с генерацией отчёта о покрытии кода.

3. Сборка и деплой. При слиянии в ветку `main` Railway автоматически собирает Docker-образ и развёртывает новую версию с нулевым временем простоя (zero-downtime deployment). Healthcheck-эндпоинт `/health` подтверждает готовность сервиса к приёму запросов.

Управление конфигурацией. Все секреты (API-ключи Yandex Cloud, токен Telegram-бота, реквизиты YooKassa) хранятся в переменных окружения Railway и загружаются через класс `Settings` на базе Pydantic BaseSettings. Исходный код не содержит конфиденциальных данных.

Инфраструктура доставки контента. Домен `pandapal.ru` обслуживается через Cloudflare, который предоставляет:
* DNS-управление с проксированием трафика через CDN;
* автоматический SSL-сертификат (TLS 1.3) для всех соединений;
* кэширование статических ресурсов фронтенда (JS, CSS, изображения) на edge-серверах;
* защиту от DDoS-атак на сетевом уровне.

Мониторинг и логирование. Модуль `bot/monitoring/` обеспечивает:
* структурированное логирование всех операций через `structlog` с выводом в формате JSON;
* отслеживание аномалий в использовании платформы (резкое увеличение запросов, нетипичные паттерны обращений);
* метрики производительности: время ответа API, количество ошибок, использование ресурсов.

Railway предоставляет встроенные инструменты мониторинга: логи в реальном времени, отслеживание потребления CPU и памяти, автоматические алерты при сбоях.

3.1. Стратегия тестирования и виды тестов

Тестирование PandaPal организовано по многоуровневой стратегии, включающей 6 категорий тестов:

1. Unit-тесты — изолированное тестирование отдельных функций и методов. Проверяют корректность логики модерации, формирования промптов, обработки ответов, игровых алгоритмов.

2. Интеграционные тесты — проверка корректного взаимодействия между модулями: связка AI-сервис ↔ модерация ↔ база данных.

3. End-to-end (e2e) тесты — полные сценарии пользовательского взаимодействия. Для фронтенда используется Playwright, для бэкенда — тестирование через API.

4. Security-тесты — проверка безопасности: SQL-инъекции, XSS, CSRF, валидация Telegram Init Data, корректность rate limiting.

5. Resilience-тесты — проверка отказоустойчивости: поведение при недоступности внешних сервисов (Yandex Cloud, Redis), корректность Circuit Breaker.

6. Performance-тесты — нагрузочное тестирование: время ответа API, пропускная способность, потребление памяти.

Тесты расположены в каталоге `tests/` с соответствующими подкаталогами. Конфигурация тестов описана в `pytest.ini` и `pyproject.toml`. Для запуска используется pytest.

2. Защита от инъекций:
   * SQLAlchemy ORM с параметризованными запросами исключает SQL-инъекции.

   * Все API-эндпоинты защищены лимитами на количество запросов в единицу времени.
   * Реализация через Redis обеспечивает распределённый rate limiting.

4. Мониторинг:
   * Модуль `bot/monitoring/` отслеживает аномалии в использовании платформы.
   * Логирование всех операций через структурированный logger (не print).

Результаты security-тестов не выявили критических уязвимостей. Дополнительно проведён статический анализ кода с помощью Bandit (отчёт: `bandit_report.json`).

Приложение В. Описание интерфейса платформы

Интерфейс PandaPal реализован как одностраничное приложение (SPA) с адаптивным дизайном для мобильных устройств. Основные экраны:

1. Главный экран (Dashboard) — отображает приветствие, быстрые действия (начать чат, открыть игры), прогресс по предметам и статус виртуального питомца.

2. AI-чат — полноэкранный чат с поддержкой Markdown-форматирования, LaTeX-формул, подсветки кода, прикрепления фотографий и записи голосовых сообщений. Ответы появляются в потоковом режиме.

3. PandaPalGo (Игры) — карусель карточек с пятью играми. Каждая игра открывается на отдельном экране с анимированным интерфейсом.

4. Моя панда — экран виртуального питомца с индикаторами настроения, сытости и энергии, кнопками действий (кормить, играть, спать).

5. Достижения — список разблокированных и заблокированных бейджей с прогресс-барами.

6. Premium — страница с описанием преимуществ подписки и кнопкой оплаты через YooKassa.

```python
def build_system_prompt(user_age: int, user_grade: int, subject: str) -> str:
    """Формирует системный промпт с учётом возраста ученика."""
    if user_grade <= 4:
        style = "простым языком, с примерами из жизни, короткими предложениями"
        emoji_policy = "используй эмодзи для наглядности"

    else:
        style = "академическим языком с научными терминами"
        emoji_policy = "минимум эмодзи, фокус на точности"

    return f"""Ты — PandaPal, дружелюбный AI-помощник для школьника {user_grade} класса.
Объясняй {style}. {emoji_policy}.
```

```python
async def moderate_message(text: str, user_age: int) -> ModerationResult:
    """Проверяет сообщение на всех уровнях модерации."""
    # Уровень 1: Фильтрация запрещённых паттернов
    if forbidden_patterns_check(text):
        return ModerationResult(allowed=False, reason="forbidden_pattern")

    # Уровень 2: Расширенная модерация (обход фильтров)
    if advanced_moderation_check(text):
        return ModerationResult(allowed=False, reason="filter_bypass")

    # Уровень 3: AI-модерация (семантический анализ)
    ai_result = await ai_moderator.check(text, user_age)
    if not ai_result.is_safe:
        return ModerationResult(allowed=False, reason="ai_moderation")

    # Уровень 4: Проверка чувствительных тем
    adult_topic = adult_topics_service.classify(text)
    if adult_topic:
        return ModerationResult(allowed=True, adult_topic=adult_topic)

    return ModerationResult(allowed=True)
```

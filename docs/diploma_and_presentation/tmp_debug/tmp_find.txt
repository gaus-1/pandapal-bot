[32] Проблема проекта
[33] Отсутствие доступной, безопасной и комплексной образовательной платформы для школьников младших и средних классов, объединяющей возможности искусственного интеллекта, интерактивного обучения и геймификации в привычной для детей среде мессенджера Telegram.
[34] Аннотация проекта
[35] В проекте разработана полнофункциональная образовательная платформа PandaPal, представляющая собой AI-помощника для школьников 1–9 классов. Платформа реализована в формате Telegram-бота и веб-приложения (Mini App) с использованием современного технологического стека: Python 3.13, aiogram 3.24, React 19, TypeScript 5, PostgreSQL 17 с расширением pgvector, Redis 7. Для генерации ответов используется YandexGPT Pro, для распознавания речи — Yandex SpeechKit, для анализа изображений — Yandex Vision OCR. Платформа включает AI-чат с адаптацией по возрасту, проверку домашних заданий по фото, голосовые вопросы, семантический поиск по базе знаний (RAG), образовательные игры и систему геймификации. Реализована многоуровневая модерация контента для обеспечения безопасности детской аудитории.
[36] География проекта
[37] Город Санкт-Петербург, с возможностью тиражирования результатов в масштабах страны через Telegram и веб-доступ. Платформа доступна по адресу https://pandapal.ru и через Telegram-бота @PandaPalBot.
[38] Целевая аудитория
[39] Школьники 1–9 классов (возраст 6–16 лет), их родители и преподаватели, заинтересованные в использовании современных технологий для повышения качества и доступности образовательного процесса.
[40] Сроки выполнения
[41] 01.10.2025 – 25.03.2026
[42] Результаты
[43] В ходе проекта разработана и развёрнута в продуктивной среде образовательная платформа PandaPal, включающая серверную часть (Python/aiogram/SQLAlchemy), клиентскую часть (React/TypeScript/Vite), интеграции с облачными сервисами Yandex Cloud (YandexGPT, SpeechKit, Vision OCR, YandexART, Embeddings API) и платёжной системой YooKassa. Платформа развёрнута на Railway с CDN через Cloudflare. Реализован 51 модуль бизнес-логики, 10 ORM-моделей, RAG-пайплайн с pgvector, 5 образовательных игр, система достижений и геймификации, многоуровневая модерация контента, Premium-подписка. Написаны автоматизированные тесты (unit, integration, e2e, security, resilience, performance).

[104] ВВЕДЕНИЕ
[105] Актуальность темы
[106] Современная система образования переживает период цифровой трансформации. Развитие технологий искусственного интеллекта, облачных вычислений и мобильных приложений расширяет возможности организации учебного процесса. По данным отчёта UNESCO «AI and Education: Guidance for Policy-makers» (2021), внедрение AI-технологий в образование помогает адаптировать учебный процесс под каждого ученика и повышать академические результаты.
[107] Отдельного внимания заслуживает разработка образовательных инструментов для школьников младших и средних классов. Эта возрастная группа активно осваивает цифровые технологии, но нуждается в адаптированном и безопасном образовательном пространстве. По данным ежегодного отчёта «Лаборатории Касперского» (2024), дети проводят в интернете в среднем 4–6 часов ежедневно, поэтому создание для них полезного и безопасного цифрового контента — задача первостепенной важности.
[108] Мессенджер Telegram насчитывает более 900 миллионов активных пользователей и давно стал привычной средой общения подростков. Telegram Bot API и технология Mini App позволяют создавать полноценные интерактивные приложения внутри мессенджера, и пользователю не приходится устанавливать отдельное приложение. Это делает Telegram удобной платформой для размещения образовательных сервисов.
[109] Однако существующие образовательные решения на базе AI часто ориентированы на взрослую аудиторию, не учитывают возрастные особенности восприятия информации и не дают необходимого уровня модерации контента для детей. Большинство таких решений ограничивается текстовым чатом и не предлагает мультимодальных возможностей: распознавание голоса, анализ изображений, визуализации, образовательные игры.
[110] Разработка образовательной платформы, объединяющей AI, мультимодальный ввод и геймификацию в безопасной среде для школьников, остаётся нерешённой задачей с высоким прикладным значением.
[111] Проблема проекта
[112] На сегодняшний день отсутствует доступная, безопасная и комплексная образовательная платформа для школьников, которая объединяла бы искусственный интеллект с интерактивными методами обучения.
[113] «Сущее» (то, что есть):
[114] –  Существующие AI-ассистенты (ChatGPT, Gemini) не адаптированы для детской аудитории и не имеют встроенной модерации контента.
[115] –  Школьные образовательные платформы не предлагают мультимодальных возможностей: голосовые вопросы, проверка домашних заданий по фото, генерация визуализаций.
[116] –  Отсутствует интеграция AI-обучения с элементами геймификации (игры, достижения, прогресс), которые способны значительно повысить мотивацию к обучению.
[117] –  Не существует образовательной платформы, доступной непосредственно в Telegram — привычной среде общения для подростков.
[118] «Должное» (то, что необходимо):
[119] –  Необходимо разработать платформу с AI-чатом, адаптирующим ответы по возрасту и классу ученика.
[120] –  Требуется реализовать мультимодальный ввод: текст, голос, фото.
[121] –  Важно обеспечить многоуровневую модерацию контента для гарантии безопасности детей.
[122] –  Необходимо внедрить элементы геймификации для повышения вовлечённости в учебный процесс.

[186] –  Риск сбоев внешних сервисов (Yandex Cloud, YooKassa). Снижается за счёт реализации паттерна Circuit Breaker, механизмов повторных попыток (retry) и уведомления пользователей о временной недоступности.
[187] –  Риск превышения лимитов API и увеличения затрат на облачные сервисы. Минимизируется через систему кэширования (Redis), rate limiting, семантический кэш RAG-запросов.
[188] –  Риск нехватки времени на полную реализацию функционала. Снижается за счёт модульной архитектуры, позволяющей последовательно наращивать функциональность, и итеративного подхода к разработке.

Такой подход позволяет находить релевантные учебные материалы не по точному совпадению слов, а по смыслу, что критически важно при работе с детской аудиторией, формулирующей вопросы нечётко и разговорным языком.
[203] Ещё одним нововведением является механизм виртуального питомца «Моя панда» — интерактивного тамагочи, состояние которого (настроение, сытость, энергия) зависит от регулярности использования платформы учеником. Питомец мотивирует ребёнка возвращаться к обучению, формируя эмоциональную привязанность и привычку ежедневных занятий.

[213] ГЛАВА 1. ТЕОРЕТИЧЕСКИЕ ОСНОВЫ РАЗРАБОТКИ ОБРАЗОВАТЕЛЬНЫХ ПЛАТФОРМ С ИСКУССТВЕННЫМ ИНТЕЛЛЕКТОМ
[214] 1.1. Искусственный интеллект в современном образовании
[215] Искусственный интеллект (ИИ) — направление компьютерных наук, занимающееся созданием систем, решающих задачи, которые ранее требовали участия человека: понимание текста, распознавание образов, принятие решений. С появлением больших языковых моделей (Large Language Models, LLM) ИИ перестал быть инструментом узких специалистов и вошёл в повседневную практику.
[216] Применение ИИ в образовании охватывает несколько ключевых направлений:
[217] 1. Персонализированное обучение. AI-системы анализируют уровень знаний учащегося, темп усвоения материала и предпочитаемый стиль подачи информации, формируя на этой основе индивидуальные образовательные траектории. В мета-анализе Zawacki-Richter et al. (2019), охватившем 146 работ, зафиксировано повышение академической успеваемости на 20–30 % при использовании персонализированных AI-инструментов.
[218] 2. Автоматизированная оценка и обратная связь. AI позволяет автоматизировать проверку заданий, предоставляя учащимся мгновенную обратную связь. Это особенно важно для предметов с однозначными ответами (математика, тесты), но современные LLM способны оценивать и творческие работы.
[219] 3. Интеллектуальные помощники и чат-боты. Образовательные чат-боты выступают в роли виртуальных репетиторов, доступных круглосуточно. Они могут отвечать на вопросы, объяснять темы, генерировать задачи и направлять учащегося в процессе изучения нового материала.
[220] 4. Мультимодальное взаимодействие. Современные AI-системы поддерживают не только текстовый ввод, но и голосовые запросы, анализ изображений (например, фото рукописного задания), генерацию визуализаций (графиков, диаграмм, схем), что обеспечивает более естественное и эффективное взаимодействие.
[221] Среди конкретных технологий, применяемых в образовательном ИИ, выделяются большие языковые модели (GPT, YandexGPT, GigaChat), системы распознавания речи (Speech-to-Text), оптического распознавания текста (OCR), генерации изображений и векторных эмбеддингов для семантического поиска.
[222] Для PandaPal выбрана модель YandexGPT — российская LLM, разработанная компанией Яндекс. Она поддерживает русский язык на высоком уровне, имеет возможность настройки параметров генерации (temperature, max_tokens) и предоставляется через облачный API Yandex Cloud, что подходит для русскоязычной образовательной платформы.
[223] 1.2. Чат-боты как инструмент обучения: обзор существующих решений
[224] Чат-бот — программа, использующая обработку естественного языка (Natural Language Processing, NLP) для ведения диалога с пользователем в текстовом или голосовом формате. В сфере образования чат-боты играют роль виртуальных репетиторов: отвечают на вопросы, разъясняют материал и помогают выстроить траекторию обучения.
[225] Различают два основных типа образовательных чат-ботов:
[226] 1. Чат-боты на основе правил (rule-based). Работают по заранее заданным сценариям и шаблонам. Способны обрабатывать ограниченный набор запросов. Примеры: боты-навигаторы по расписанию, боты с тестами.
[227] 2. Чат-боты на основе AI (AI-powered). Используют большие языковые модели для генерации ответов. Способны вести свободный диалог, понимать контекст, адаптировать стиль общения. Примеры: ChatGPT, Claude, Gemini, YandexGPT.
[228] Обзор существующих образовательных платформ с AI:
[229] –  ChatGPT (OpenAI) — мощная языковая модель, широко используемая для образовательных целей. Недостатки: не адаптирована для детей, отсутствие встроенной модерации для детского контента, нет интеграции с Telegram, ориентация на англоязычную аудиторию.
[230] –  Duolingo — платформа для изучения языков с элементами геймификации. Достоинства: эффективная геймификация, персонализация. Недостатки: ограничена одним предметом (иностранные языки), не использует чат-формат.
[231] –  Учи.ру — российская образовательная платформа для школьников. Достоинства: адаптация под российскую программу. Недостатки: ограниченные AI-возможности, отсутствие мультимодального ввода.
[232] –  Яндекс Учебник — образовательные материалы от Яндекса. Достоинства: качественный контент, интеграция с российской программой. Недостатки: не является чат-ботом, нет голосового ввода и анализа фото.
[233] Платформа PandaPal объединяет преимущества перечисленных решений: AI-чат с адаптацией по возрасту (как ChatGPT, но безопаснее), геймификация (как Duolingo), ориентация на российскую программу (как Учи.ру) и интеграция с экосистемой Yandex Cloud. При этом PandaPal размещается непосредственно в Telegram, что упрощает доступ для целевой аудитории.
[234] Согласно обзору Hwang & Chang (2021), чат-боты в образовании повышают мотивацию учащихся и снижают тревожность при обращении за помощью, одновременно обеспечивая круглосуточную доступность ресурса. При этом авторы подчёркивают вспомогательную роль ИИ: AI должен поддерживать обучение, а не подменять его.
[235] 1.3. Технология Retrieval-Augmented Generation (RAG) и её применение в образовании
[236] Retrieval-Augmented Generation (RAG) — архитектурный паттерн, при котором LLM дополняется результатами поиска по внешней базе знаний. RAG-система сначала находит релевантные документы, а затем передаёт их модели как контекст для генерации ответа, благодаря чему модель опирается не только на обучающие данные, но и на актуальные источники.
[237] Архитектура RAG включает три основных компонента:
[238] 1. Хранилище знаний. Коллекция документов, статей, учебных материалов, индексированных для эффективного поиска. В PandaPal используется PostgreSQL с расширением pgvector, позволяющим хранить и искать векторные представления текстов.
[239] 2. Модель эмбеддингов. Нейросетевая модель, преобразующая текстовые фрагменты в числовые векторы (эмбеддинги), которые отражают семантическое значение текста. В PandaPal используется Yandex Embeddings API.
[240] 3. Генеративная модель. Большая языковая модель, получающая найденные документы как контекст и генерирующая ответ на основе этого контекста. В PandaPal используется YandexGPT Pro.
[241] Процесс работы RAG в PandaPal:
[242] 1. Пользователь задаёт вопрос (например, «Что такое фотосинтез?»).
[243] 2. Вопрос преобразуется в вектор с помощью Yandex Embeddings API.
[244] 3. Выполняется поиск ближайших по смыслу документов в таблице knowledge_embeddings (PostgreSQL + pgvector) с использованием косинусного расстояния.
[245] 4. Найденные фрагменты передаются вместе с вопросом в YandexGPT Pro.
[246] 5. Модель генерирует ответ, опираясь на достоверные данные из базы знаний.
[247] Преимущества RAG в образовательном контексте:
[248] –  Снижение галлюцинаций. Модель опирается на проверенные источники, а не генерирует факты произвольно. В образовательном контексте это особенно существенно.
[249] –  Актуальность информации. Базу знаний можно обновлять без переобучения модели.
[250] –  Прозрачность. Ответ может содержать ссылки на конкретные источники из базы знаний.
[251] –  Контролируемость. Администратор решает, какие материалы попадают в базу.
[252] В PandaPal RAG-пайплайн дополнен несколькими оптимизациями:
[253] –  Расширение запросов (Query Expansion) — автоматическое обогащение пользовательского запроса синонимами и связанными терминами для повышения полноты поиска.
[254] –  Семантический кэш — кэширование результатов поиска для семантически похожих запросов, что снижает нагрузку на API эмбеддингов.
[255] –  Компрессия контекста — сжатие найденных документов для оптимального использования контекстного окна LLM.
[256] –  Ранжирование результатов (Reranker) — повторное ранжирование найденных документов по релевантности перед передачей в модель.
[257] 1.4. Геймификация учебного процесса: теория и практика
[258] Геймификация (gamification) — это применение элементов игрового дизайна в неигровых контекстах с целью повышения вовлечённости, мотивации и активности пользователей. В образовании геймификация предполагает внедрение таких элементов, как баллы (XP), уровни, достижения (бейджи), рейтинги и соревнования в учебный процесс.
[259] Теоретические основы геймификации опираются на ряд психологических теорий:
[260] 1. Теория самодетерминации (Self-Determination Theory, Deci & Ryan). Выделяет три базовые потребности человека: автономия, компетентность и связанность. Геймификация удовлетворяет все три: пользователь сам выбирает задачи (автономия), получает обратную связь о своём прогрессе (компетентность), может видеть рейтинги других (связанность).
[261] 2. Теория потока (Flow Theory, Csikszentmihalyi). Описывает состояние максимальной вовлечённости, возникающее при балансе между сложностью задачи и навыками пользователя. Геймификация способствует достижению потокового состояния через адаптивную сложность.
[262] 3. Теория подкрепления (Reinforcement Theory, Skinner). Положительное подкрепление (награды, баллы, достижения) закрепляет желаемое поведение — в данном случае активное обучение.
[263] В мета-анализе Sailer & Homner (2020) показано, что геймификация увеличивает вовлечённость учащихся в среднем на 25 %. В отечественном исследовании Кречетовой и др. (2023) средний балл учащихся вырос с 65,4 до 78,9 после внедрения геймифицированных элементов, а 92 % педагогов отметили повышение мотивации.
[264] В PandaPal система геймификации реализована через несколько механизмов:
[265] –  Очки опыта (XP) — начисляются за отправку сообщений, решение задач, участие в играх.
[266] –  Достижения — система из более чем 15 бейджей, разблокируемых при достижении определённых показателей (первое сообщение, 100 сообщений, неделя подряд, вопросы по пяти и более предметам).
[267] –  Образовательные игры PandaPalGo — (5 игр: тамагочи «Моя панда», крестики-нолики, шашки, 2048, эрудит), развивающие логику, стратегическое мышление и словарный запас.
[268] –  «Моя панда» — виртуальный питомец (тамагочи), который меняет настроение и активность в зависимости от того, насколько регулярно ученик пользуется платформой. Этот механизм создаёт эмоциональную привязанность и мотивирует к ежедневному обучению.
[269] –  Прогресс и статистика — пользователь видит свой прогресс по предметам, количество решённых задач, дни активности.
[270] 1.5. Безопасность детей в цифровой среде: модерация и фильтрация контента
[271] Информационная безопасность детей в интернете — вопрос, требующий системного подхода. В Российской Федерации данная сфера регулируется Федеральным законом № 436-ФЗ «О защите детей от информации, причиняющей вред их здоровью и развитию», который устанавливает требования к контентной фильтрации в образовательных учреждениях и на платформах, ориентированных на несовершеннолетних пользователей.
[272] При разработке образовательной платформы с AI для детей особое внимание необходимо уделить следующим аспектам безопасности:
[273] 1. Фильтрация входящего контента. Пользовательские запросы (текст, голос, изображения) должны проверяться на наличие недопустимого содержания до обработки AI-моделью. Тем самым предотвращаются попытки получить небезопасный контент через манипуляцию промптами (prompt injection).
[274] 2. Модерация выходного контента. Ответы AI-модели также требуют проверки, поскольку языковые модели могут генерировать неуместную или некорректную информацию. Многоуровневая фильтрация на выходе добавляет дополнительный рубеж защиты.
[275] 3. Адаптация контента по возрасту. Ответы должны учитывать возраст и класс ученика, используя соответствующую лексику, сложность объяснений и примеры.
[276] В PandaPal модерация контента реализована как многоуровневая система:
[277] –  Уровень 1: Фильтрация запрещённых паттернов (forbidden patterns). Проверка входящих сообщений на наличие нецензурной лексики, запрещённых тем и опасного контента с использованием регулярных выражений и словарей.
[278] –  Уровень 2: Расширенная модерация (advanced moderation). Анализ контекста сообщения для выявления скрытых попыток обхода фильтров, использование эвристик и лингвистического анализа.
[279] –  Уровень 3: AI-модерация. Использование языковой модели для семантического анализа сообщений, которые прошли первые два уровня, но могут содержать неуместный контент в неявной форме.
[280] –  Уровень 4: Сервис взрослых тем (adult topics service). Специализированный модуль для корректной обработки вопросов на чувствительные темы (здоровье, семья), которые не являются запрещёнными, но требуют деликатного подхода с учётом возраста.
[281] –  Адаптивный промпт. Системный промпт для YandexGPT содержит инструкции по адаптации ответов в зависимости от возраста и класса ученика, запрет на определённые темы и указание стилистических характеристик ответов.

[351] 1. Модульная структура сервисов. [352] 2. Слой ORM-моделей. Каталог bot/models/ содержит 10 моделей SQLAlchemy, описывающих сущности предметной области: User (пользователь), Chat (сообщения чата), Learning (учебные сессии), Games (игровые сессии), Payments (платёжные транзакции), PandaPet (виртуальный питомец), Referral (реферальная программа), Analytics (аналитические данные).
[353] 3. REST API. Каталог bot/api/ содержит эндпоинты для Mini App и веб-приложения, следуя RESTful-принципам. Все защищённые эндпоинты валидируют владельца ресурса через заголовок X-Telegram-Init-Data.
[354] 4. Middleware и безопасность. Каталоги bot/middleware/ и bot/security/ реализуют сквозные механизмы: rate limiting, аудит запросов, криптографическая проверка Telegram-данных, валидация входных данных через Pydantic v2.

[404] 2.3. Интеграция с YandexGPT и формирование AI-ответов
[405] Ключевая интеграция платформы — взаимодействие с YandexGPT, LLM от Яндекса. Для взаимодействия с моделью реализован сервис yandex_cloud_service.py (~43 КБ), который инкапсулирует всю логику работы с Yandex Cloud API.
[406] Формирование AI-ответа включает несколько этапов:
[407] 1. Построение промпта (prompt_builder.py, ~16 КБ). Системный промпт формируется динамически в зависимости от:
[408] –  Возраста и класса пользователя — определяет сложность языка и объём объяснений.
[409] –  Выбранного предмета — добавляет предметно-специфические инструкции.
[410] –  Контекста предыдущих сообщений — обеспечивает связность диалога.
[411] –  Результатов RAG-поиска — включает релевантные фрагменты из базы знаний.
[412] 2. Формирование контекста (ai_context_builder.py, ~12 КБ). Модуль управляет историей диалога, ограничивая её 200 сообщениями. При достижении 400 сообщений автоматически удаляются старейшие записи. История помогает модели поддерживать связность ответов.
[413] 3. Генерация ответа. Запрос отправляется к YandexGPT Pro через REST API Yandex Cloud с параметрами: temperature=0.35 (низкая «температура» для более точных ответов), max_tokens=8192.
[414] 4. Обработка ответа (yandex_ai_response_generator.py, ~22 КБ). Модуль поддерживает два режима генерации:
[415] –  Обычный (non-streaming) — ответ получается целиком после завершения генерации.
[416] –  Потоковый (streaming, SSE) — ответ передаётся по частям по мере генерации, обеспечивая мгновенную обратную связь пользователю.
[417] 5. Постобработка ответа. Включает очистку от нежелательных артефактов (response_cleaner/), проверку формул (formula_explainer.py), генерацию визуализаций при необходимости (visualization_service.py).
[418] Помимо YandexGPT, платформа интегрирована с другими сервисами Yandex Cloud:
[419] –  SpeechKit STT (speech_service.py) — распознаёт голосовые сообщения пользователей, преобразуя их в текст для последующей обработки AI.
[420] –  Vision OCR (vision_service.py) — распознаёт текст на фотографиях, что позволяет проверять домашние задания по фото.
[421] –  YandexART (yandex_art_service.py) — генерирует изображения по текстовому описанию для визуализации учебных тем.
[422] 2.4. RAG-пайплайн: семантический поиск по базе знаний
[423] RAG-пайплайн PandaPal реализован в каталоге bot/services/rag/ и состоит из пяти основных компонентов:
[424] 1. Векторный поиск (vector_search.py, ~8.5 КБ). Выполняет поиск ближайших векторов в таблице knowledge_embeddings с использованием pgvector. Применяет косинусное расстояние для определения семантической близости. Возвращает top-k наиболее релевантных документов.
[425] 2. Расширение запросов (query_expander.py, ~26 КБ). Автоматически обогащает пользовательский запрос синонимами, связанными терминами и альтернативными формулировками. Приём повышает полноту поиска — в первую очередь для коротких и неточных запросов, типичных для детской аудитории.
[426] 3. Ранжирование (reranker.py, ~7.7 КБ). После первичного векторного поиска выполняет повторное ранжирование результатов по релевантности, учитывая не только семантическую близость, но и контекстуальные факторы.
[427] 4. Компрессия контекста (compressor.py, ~4.2 КБ). Сжимает найденные документы до оптимального размера для контекстного окна LLM, удаляя избыточную информацию и сохраняя ключевые факты.
[428] 5. Семантический кэш (semantic_cache.py, ~7.7 КБ). Кэширует результаты поиска для семантически похожих запросов. Если новый запрос похож на предыдущий (по косинусному расстоянию эмбеддингов), результат возвращается из кэша без обращения к базе данных, что снижает нагрузку и ускоряет ответ.
[429] Наполнение базы знаний осуществляется через скрипт scripts/update_knowledge_base.py, который индексирует учебные материалы и энциклопедические источники, преобразуя их в эмбеддинги через Yandex Embeddings API и сохраняя в таблицу knowledge_embeddings.

[462] 2.6. Система модерации контента
[463] Модерация в PandaPal устроена как многоуровневый конвейер: каждое сообщение проходит несколько ступеней проверки. Основные модули:
[464] –  moderation_service.py (~20 КБ) — основной сервис базовой модерации. Содержит словари запрещённых паттернов, регулярные выражения для обнаружения нецензурной лексики и опасных тем.
[465] –  advanced_moderation.py (~21 КБ) — расширенная модерация. Анализирует контекст сообщения, выявляет попытки обхода фильтров (замена букв символами, транслитерация, использование эвфемизмов).
[466] –  ai_moderator.py (~2.4 КБ) — AI-модерация. Использует YandexGPT для семантического анализа сообщений, которые прошли предыдущие уровни, но могут содержать неуместный контент.
[467] –  adult_topics_service.py (~7.4 КБ) — обработка чувствительных тем. Определяет вопросы на деликатные темы (здоровье, семейные отношения) и формирует соответствующие инструкции для AI, обеспечивая корректный и деликатный ответ с учётом возраста.
[468] Результат модерации определяет дальнейшую обработку сообщения: допуск к AI-обработке, блокировка с уведомлением пользователя или передача администратору для ручной проверки.
[469] 2.7. Образовательные игры и геймификация
[470] В PandaPal реализовано 5 образовательных игр, объединённых под брендом PandaPalGo:
[471] 1. «Моя панда» (тамагочи) — виртуальный питомец-панда, которого нужно кормить, укладывать спать, играть с ним. Состояние панды зависит от регулярности использования платформы. Реализация: фронтенд MyPanda/, бэкенд panda_pet_service.py (~14 КБ), эндпоинты panda_pet_endpoints.py (~9 КБ).
[472] 2. Крестики-нолики (TicTacToe) — классическая игра с AI-противником. Реализация: TicTacToe.tsx (~10 КБ), серверная логика в game_engines/.
[473] 3. Шашки (Checkers) — стратегическая настольная игра с AI-противником, несколько уровней сложности. Реализация: Checkers.tsx (~16 КБ).
[474] 4. 2048 — числовая головоломка, развивающая логическое мышление. Реализация: Game2048.tsx (~16 КБ).
[475] 5. Эрудит (Erudite) — словесная игра, расширяющая словарный запас. Использует словарь из erudite_dictionary.py (~14 КБ). Реализация: Erudite.tsx (~13 КБ).
[476] Серверная логика игр реализована в bot/services/games_service/ и bot/services/game_engines/, а API-эндпоинты — в bot/api/games_endpoints.py (~21 КБ). Все игровые сессии сохраняются в базе данных с привязкой к пользователю, что позволяет отслеживать прогресс и начислять XP.
[477] Система геймификации (gamification_service.py, ~31 КБ) управляет:
[478] –  Начислением XP за различные действия (сообщения, решённые задачи, победы в играх).
[479] –  Разблокировкой достижений при достижении определённых условий.
[480] –  Расчётом уровня пользователя на основе накопленного XP.
[481] –  Формированием статистики прогресса для отображения на дашборде.
[482] 2.8. Premium-подписка и платёжная система
[483] PandaPal предлагает Premium-подписку стоимостью 299 ₽/месяц, расширяющую возможности платформы: неограниченное количество AI-запросов, приоритетная обработка, расширенные визуализации, дополнительный контент по предметам.
[484] Интеграция с платёжной системой YooKassa реализована через несколько модулей:
[485] –  payment_service.py (~19 КБ) — основной сервис обработки платежей. Создание платёжных ссылок, обработка webhook-уведомлений от YooKassa, обновление статуса подписки.
[486] –  subscription_service.py (~11 КБ) — управление подписками. Активация, продление, отмена подписки, проверка статуса.
[487] –  recurring_payment_service.py (~7 КБ) — автоматическое продление подписки через сохранённые платёжные методы.
[488] –  premium_features_service.py (~20 КБ) — логика Premium-функций. Определение доступного функционала в зависимости от статуса подписки.
[489] –  premium_endpoints.py (~32 КБ) — REST API для работы с Premium из Mini App.
[490] Дополнительно реализована реферальная программа (referral_service.py): преподаватели и партнёры получают персональную ссылку, и оплаты, совершённые по этой ссылке, учитываются в системе выплат.
[491] 2.9. Развёртывание и обеспечение непрерывной доставки
[492] Платформа PandaPal развёрнута в продуктивной среде на платформе Railway — PaaS-решении, поддерживающем контейнеризацию через Docker, управляемые базы данных и автоматическое масштабирование. Выбор Railway обусловлен простотой деплоя, встроенной поддержкой переменных окружения и отсутствием необходимости в ручном администрировании серверов.
[493] Контейнеризация и Dockerfile. Серверная часть упакована в Docker-контейнер на основе образа python:3.13-slim. Dockerfile включает установку системных зависимостей, копирование исходного кода, установку Python-зависимостей через pip и определение точки входа (web_server.py). Многоступенчатая сборка минимизирует размер финального образа.
[494] Конвейер CI/CD реализован через GitHub Actions и включает следующие этапы:
[495] 1. Линтинг и статический анализ. При каждом push и pull request автоматически запускаются проверки: ESLint и TypeScript strict mode для фронтенда, Bandit для анализа безопасности Python-кода.
[496] 2. Автоматизированное тестирование. Запускаются unit-тесты и интеграционные тесты с генерацией отчёта о покрытии кода.
[497] 3. Сборка и деплой. При слиянии в ветку main Railway автоматически собирает Docker-образ и развёртывает новую версию с нулевым временем простоя (zero-downtime deployment). Healthcheck-эндпоинт /health подтверждает готовность сервиса к приёму запросов.
[498] Управление конфигурацией. Все секреты (API-ключи Yandex Cloud, токен Telegram-бота, реквизиты YooKassa) хранятся в переменных окружения Railway и загружаются через класс Settings на базе Pydantic BaseSettings. Исходный код не содержит конфиденциальных данных.
[499] Инфраструктура доставки контента. Домен pandapal.ru обслуживается через Cloudflare, который предоставляет:
[500] –  DNS-управление с проксированием трафика через CDN;
[501] –  автоматический SSL-сертификат (TLS 1.3) для всех соединений;
[502] –  кэширование статических ресурсов фронтенда (JS, CSS, изображения) на edge-серверах;
[503] –  защиту от DDoS-атак на сетевом уровне.
[504] Мониторинг и логирование. Модуль bot/monitoring/ обеспечивает:
[505] –  структурированное логирование всех операций через structlog с выводом в формате JSON;
[506] –  отслеживание аномалий в использовании платформы (резкое увеличение запросов, нетипичные паттерны обращений);
[507] –  метрики производительности: время ответа API, количество ошибок, использование ресурсов.
[508] Railway предоставляет встроенные инструменты мониторинга: логи в реальном времени, отслеживание потребления CPU и памяти, автоматические алерты при сбоях.

[510] ГЛАВА 3. ИССЛЕДОВАТЕЛЬСКАЯ ЧАСТЬ: ТЕСТИРОВАНИЕ, АНАЛИЗ И РЕЗУЛЬТАТЫ
[511] 3.1. Стратегия тестирования и виды тестов
[512] Тестирование PandaPal организовано по многоуровневой стратегии, включающей 6 категорий тестов:
[513] 1. Unit-тесты — изолированное тестирование отдельных функций и методов. Проверяют корректность логики модерации, формирования промптов, обработки ответов, игровых алгоритмов.
[514] 2. Интеграционные тесты — проверка корректного взаимодействия между модулями: связка AI-сервис ↔ модерация ↔ база данных.
[515] 3. End-to-end (e2e) тесты — полные сценарии пользовательского взаимодействия. Для фронтенда используется Playwright, для бэкенда — тестирование через API.
[516] 4. Security-тесты — проверка безопасности: SQL-инъекции, XSS, CSRF, валидация Telegram Init Data, корректность rate limiting.
[517] 5. Resilience-тесты — проверка отказоустойчивости: поведение при недоступности внешних сервисов (Yandex Cloud, Redis), корректность Circuit Breaker.
[518] 6. Performance-тесты — нагрузочное тестирование: время ответа API, пропускная способность, потребление памяти.
[519] Тесты расположены в каталоге tests/ с соответствующими подкаталогами. Конфигурация тестов описана в pytest.ini и pyproject.toml. Для запуска используется pytest.

[524] 3.3. Анализ безопасности платформы
[525] Безопасность PandaPal обеспечивается на нескольких уровнях:
[526] 1. Аутентификация и авторизация:
[527] –  Все защищённые API-эндпоинты валидируют заголовок X-Telegram-Init-Data, содержащий криптографически подписанные данные пользователя от Telegram.
[528] –  Проверка владельца ресурса: пользователь может получить доступ только к своим данным (профиль, история, прогресс).
[529] 2. Защита от инъекций:
[530] –  SQLAlchemy ORM с параметризованными запросами исключает SQL-инъекции.
[531] –  Pydantic v2 валидация входных данных предотвращает передачу некорректных типов данных.
[532] –  Автоматическое экранирование пользовательского ввода в промптах.
[533] 3. Rate Limiting:
[534] –  Все API-эндпоинты защищены лимитами на количество запросов в единицу времени.
[535] –  Реализация через Redis обеспечивает распределённый rate limiting.
[536] 4. Мониторинг:
[537] –  Модуль bot/monitoring/ отслеживает аномалии в использовании платформы.
[538] –  Логирование всех операций через структурированный logger (не print).
[539] Результаты security-тестов не выявили критических уязвимостей. Дополнительно проведён статический анализ кода с помощью Bandit (отчёт: bandit_report.json).

[687] Приложение В. Описание интерфейса платформы
[688] Интерфейс PandaPal реализован как одностраничное приложение (SPA) с адаптивным дизайном для мобильных устройств. Основные экраны:
[689] 1. Главный экран (Dashboard) — отображает приветствие, быстрые действия (начать чат, открыть игры), прогресс по предметам и статус виртуального питомца.
[690] 2. AI-чат — полноэкранный чат с поддержкой Markdown-форматирования, LaTeX-формул, подсветки кода, прикрепления фотографий и записи голосовых сообщений. Ответы появляются в потоковом режиме.
[691] 3. PandaPalGo (Игры) — карусель карточек с пятью играми. Каждая игра открывается на отдельном экране с анимированным интерфейсом.
[692] 4. Моя панда — экран виртуального питомца с индикаторами настроения, сытости и энергии, кнопками действий (кормить, играть, спать).
[693] 5. Достижения — список разблокированных и заблокированных бейджей с прогресс-барами.
[694] 6. Premium — страница с описанием преимуществ подписки и кнопкой оплаты через YooKassa.

[789] Архитектура программного комплекса
[790] Архитектура системы построена по паттерну «Асинхронный микромонолит» и состоит из следующих слоев:

1. КЛИЕНТСКИЙ СЛОЙ (Frontend)
   — Telegram Bot (aiogram 3): мессенджер-интерфейс
   — Telegram Mini App (React 19, Zustand): встроенное веб-приложение
   — Landing Page (React, Tailwind CSS): промо-сайт

2. СЕРВЕРНЫЙ СЛОЙ (Backend)
   — Entrypoint: aiohttp web-сервер (обработка Webhook)
   — Security / Middleware: 4-уровневый AI модератор (RegExp, Levenshtein, ML-Classifier)
   — Business Logic Services: Game Engine, RAG Engine, Payments API

3. СЛОЙ ДАННЫХ (Databases)
   — Основная БД: PostgreSQL 17 + расширение pgvector (индексы HNSW)
   — Быстрый кэш и стейты: Redis 7.1 (In-Memory)

4. ВНЕШНИЕ ИНТЕГРАЦИИ (External APIs)
   — Yandex Cloud (SpeechKit STT, Vision OCR, YandexGPT Pro)
   — YooKassa API (вебхуки для обработки платежей)

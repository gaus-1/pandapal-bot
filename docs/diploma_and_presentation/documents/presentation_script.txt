# Сценарий предзащиты PandaPal (УЛЬТИМАТИВНАЯ ТЕХНИЧЕСКАЯ ВЕРСИЯ)

Этот текст максимально насыщен всеми технологиями, архитектурными паттернами и фишками бэкенда, фронтенда, тестирования и деплоя. Вы будете звучать как Senior Tech Lead.

---

## ЧАСТЬ 1: ВЕБСАЙТ (Фронтенд, Архитектура, Безопасность)

*(На видео: прокрутка сайта, секций, футера и нажатие главной кнопки)*

**СОПРОВОЖДЕНИЕ:**
«Добрый день. Представляю проект PandaPal — высоконагруженную образовательную платформу и AI-робота для школьников.
На видео показан веб-сайт проекта — наша визитная карточка, реализованная на самом современном фронтенд-стеке: **React 19, TypeScript 5, сборщик Vite и фреймворк Tailwind CSS 3**. Веб-сайт написан с использованием компонентного подхода и механизма Lazy Loading для максимальной оптимизации загрузки (First Contentful Paint).

Мы реализовали адаптивный UI (Responsive Web Design) с поддержкой темных и светлых тем. На сайте размещены динамические блоки для родителей, объясняющие логику работы системы.
Особое внимание уделено юридической чистоте: в футере размещены Оферта, Политика Конфиденциальности и Cookie-баннер, соответствующие требованиям РКН. Безопасность и маршрутизация сайта защищены **Cloudflare** (SSL и CDN защитой).
При нажатии главной кнопки срабатывает умный роутинг (Deep Linking `tg://resolve`), который бесшовно прокидывает пользователя прямо в ядро продукта — Telegram Mini App, куда мы сейчас и перейдем».

## ЧАСТЬ 2: TELEGRAM MINI APP, ИГРЫ И ПЛАТЕЖИ (Стейт-менеджемент и Redis)

*(На видео: Mini App, Дашборд, Игры (PandaPalGo), Тамагочи, Premium)*

**СОПРОВОЖДЕНИЕ:**
«Мы перешли в Telegram Mini App. Это полноценное Single Page Application, загружающееся внутри мессенджера (Telegram Web Apps SDK 8.0). Защита приложения начинается еще на входе: бэкенд строго валидирует подпись `initData` по алгоритму HMAC-SHA-256 через `X-Telegram-Init-Data`, что исключает подделку сессии пользователя. Для управления состояниями на фронтенде мы используем **Zustand 5**, а для кэширования запросов к API — **TanStack Query 5**.

Во вкладке **Достижения** находится дашборд ученика: бэкенд на Python (FastAPI/aiohttp) вычисляет кривую прогресса в реальном времени. Данные извлекаются из БД **PostgreSQL 17** через асинхронный ORM **SQLAlchemy 2.0 (`asyncpg`)**.

Во вкладке **PandaPalGo** мы реализовали мощный движок геймификации: Шашки, 2048, Эрудит и симуляцию виртуального питомца "Моя Панда" (Тамагочи). Показатели питомца пересчитываются на сервере через математические Time Deltas, а стейты (FSM-машины состояний) переброшены в **Redis 7 (Upstash)**. Эта In-Memory БД обеспечивает моментальный I/O отклик для игр, снимая всю нагрузку с основной дисковой базы.

Отдельная гордость — вкладка **Premium и Финансы**. Мы интегрировали биллинговую систему по официальному API **ЮKassa**. Фронтенд собирает форму, а наш бэкенд через асинхронные Webhook-эндпоинты безопасно обрабатывает коллбэки об успешной оплате, с автоматическим продлением (Recurring Payments) подписки, при этом строго изолируя банковские данные. Параллельно введена Реферальная программа со сложным учетом конверсий и выплат.»

## ЧАСТЬ 3: БЭКЕНД, RAG, МОДЕРАЦИЯ И СТРИМИНГ (AI-Сервисы)

*(На видео: Ввод сообщений в Чат, отправка фото, отправка ГС, стриминг ответа, прокрутка истории)*

**СОПРОВОЖДЕНИЕ:**
«А теперь самое главное — архитектура AI-ядра. На видео демонстрируется флоу работы Чата. Наш бэкенд спроектирован как **Асинхронный Микромонолит** (`aiogram 3`, Dependency Injection). Смотрите, что происходит "под капотом":

1. **Мультимодальность:** Пользователь отправляет аудио или фото задачи. Бэкенд на лету вызывает **Yandex Vision (OCR)** для вытаскивания текста с картинок и **SpeechKit (STT)** для транскрибации голоса.
2. **Безопасность (AI-Модерация):** Прежде чем пустить запрос к AI, он валидируется через **Pydantic v2**, проходит Token Bucket Rate Limiting (от DDoSa), а затем пускается в наш 4-уровневый фильтр безопасности. Если система по паттернам или пре-LLM классификатору распознает политику или насилие, запрос жестко блокируется, срабатывает защитный экран `Emergency` на клиенте.
3. **Векторная БД и RAG:** Текст пользователя конвертируется в эмбеддинги (Embeddings API). Затем в PostgreSQL, применяя расширение **`pgvector`** и индексы **HNSW**, мы реализовываем сложный математический поиск по сходству векторов (Cosine Similarity). Бэкенд достает нужный академический абзац из базы знаний и подает его нейросети как жесткий контекст (RAG пайплайн). Дополнительно работает `Semantic Cache` для экономии ресурсов на одинаковых вопросах.
4. **Стриминг и Генерация:** Итоговый промпт загружается в **YandexGPT Pro**. Для идеального UX ответ отдается на клиент по протоколу **Server-Sent Events (SSE)**. Текст печатается порциями в реальном времени. Если есть математика, бэкенд генерирует графики (Python Matplotlib) прямо в чат, а для визуализации — вызывает **YandexART**, собирая мультимодальный ответ.

## ЧАСТЬ 4: CI/CD, ТЕСТИРОВАНИЕ И ДЕПЛОЙ (Production-Ready)

*(Видео может плавно заканчиваться или зацикливаться)*

**СОПРОВОЖДЕНИЕ:**
«В завершение отмечу инженерные стандарты качества проекта. Внедрена **Continuous Integration (CI/CD)** через **GitHub Actions**.
У нас написана пирамида автотестов: от юнит и интеграционных на **Pytest**, до проверок отказоустойчивости (Resilience tests) и End-to-End тестов интерфейса средствами **Playwright**.
Каждый коммит автоматически проходит статический анализ (flake8/eslint) и аудит безопасности утилитами **Pip-audit и Bandit**.

Авторизованный деплой выполняется по принципу Zero-Downtime на облачную PaaS-платформу **Railway**, где все сервисы изолированы через Docker-контейнеры, а миграции схем БД накатываются автоматически через **Alembic**.
Проект PandaPal полностью Production-Ready, имеет высочайшую отказоустойчивость и готов к масштабированию с первого дня».

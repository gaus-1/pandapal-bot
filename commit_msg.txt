refactor: split large modules for SOLID SRP compliance

Major structural refactoring to address Single Responsibility Principle
violations across 6 large files. All public APIs preserved via
backward-compatible re-exports. Zero new test failures.

Changes by module:

1. bot/services/adult_topics_service.py (919 -> 190 lines)
   - Extract 26 topic definitions into bot/config/adult_topics_data.py
   - Service now loads data from config, keeping only logic methods

2. bot/services/visualization/detector.py (1809 -> 299 lines)
   - Split 1604-line detect() into 6 sub-detector modules:
     request_words.py, schemes.py, diagrams.py, maps.py,
     physics.py, math_graphs.py
   - detect() is now an ~80-line orchestrator
   - Refactored existing tables_and_diagrams.py

3. bot/database.py (633 lines) -> bot/database/ package

   - alembic_utils.py: Alembic config and migration logic
   - sql_migrations.py: SQL migrations (premium, payments)
   - service.py: DatabaseService class
   - __init__.py: re-exports + init_database() coordinator

4. bot/handlers/ai_chat/text.py (775 -> 509 lines)
   - Extract 6 helpers from handle_ai_message() into helpers.py:
     handle_image_generation_request, detect_and_translate_message,
     build_visualization_enhanced_message, clean_auto_generation_mentions,
     send_ai_response, offer_feedback_form

5. bot/services/cache_service.py (652 lines) -> bot/services/cache/ package
   - memory.py: CacheConfig + MemoryCache
   - service.py: CacheService + singleton + cached decorator
